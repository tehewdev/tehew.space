📚 Sobre o Bubble Sort

Complexidade: O(n²) no pior caso | O(n) no melhor caso | O(n²) médio

Como funciona: Compara elementos adjacentes e os troca se estiverem na ordem errada. O maior elemento "borbulha" para o final em cada passagem.

Vantagens: Simples de implementar, estável, detecta arrays já ordenados.

Desvantagens: Muito lento para arrays grandes.

💻 Implementação em JavaScript

// Bubble Sort - Algoritmo de Ordenação function bubbleSort(arr) { let n = arr.length; for (let i = 0; i < n; i++) { for (let j = 0; j < n - i - 1; j++) { if (arr[j] > arr[j + 1]) { // Troca os elementos [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]]; } } } return arr; }

🌳 Árvore Binária de Busca (BST)

Uma estrutura hierárquica onde cada nó tem no máximo dois filhos. O filho esquerdo é sempre menor e o direito sempre maior que o nó pai.

📊 Complexidades da BST

Busca: O(log n) balanceada | O(n) desbalanceada

Inserção: O(log n) balanceada | O(n) desbalanceada

Remoção: O(log n) balanceada | O(n) desbalanceada

💻 Implementação de Inserção

class Node { constructor(value) { this.value = value; this.left = null; this.right = null; } } function insert(root, value) { if (root === null) { return new Node(value); } if (value < root.value) { root.left = insert(root.left, value); } else if (value > root.value) { root.right = insert(root.right, value); } return root; }

🕸️ Grafos e Algoritmos de Busca

Grafos representam relações entre objetos. Visualize algoritmos de busca em largura (BFS) e profundidade (DFS).

🎯 Algoritmos de Busca

BFS (Breadth-First Search): Explora todos os vizinhos antes de ir mais fundo. Usa uma fila. Complexidade: O(V + E)

DFS (Depth-First Search): Explora o máximo de profundidade antes de retroceder. Usa uma pilha. Complexidade: O(V + E)

Dijkstra: Encontra o caminho mais curto entre dois vértices. Complexidade: O((V + E) log V)

🔗 Lista Ligada (Linked List)

Estrutura de dados dinâmica onde cada elemento (nó) contém um valor e uma referência para o próximo nó.

📊 Complexidades

Inserir no início: O(1) | Inserir no fim: O(n) sem tail, O(1) com tail

Buscar: O(n) | Deletar: O(n)

💻 Implementação

class ListNode { constructor(value) { this.value = value; this.next = null; } } class LinkedList { constructor() { this.head = null; this.size = 0; } prepend(value) { const newNode = new ListNode(value); newNode.next = this.head; this.head = newNode; this.size++; } }

📚 Stack (Pilha) & Queue (Fila)

Estruturas lineares com regras específicas de inserção e remoção.

📚 Stack (LIFO)

Last In, First Out: O último a entrar é o primeiro a sair.

Complexidade: Push O(1) | Pop O(1) | Peek O(1)

🚶 Queue (FIFO)

First In, First Out: O primeiro a entrar é o primeiro a sair.

Complexidade: Enqueue O(1) | Dequeue O(1) | Front O(1)

📈 Notação Big O Explicada

O(1) - Constante: Tempo fixo, não depende do tamanho da entrada.

O(log n) - Logarítmica: Divide o problema pela metade a cada iteração.

O(n) - Linear: Tempo cresce proporcionalmente ao tamanho da entrada.

O(n log n) - Linearítmica: Comum em algoritmos eficientes de ordenação.

O(n²) - Quadrática: Loops aninhados, cresce rapidamente.
